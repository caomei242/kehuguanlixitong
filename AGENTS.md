# Codex Project Notes

- 本项目的 Obsidian 项目根目录固定为 `/Users/gd/Library/Mobile Documents/iCloud~md~obsidian/Documents/项目管理/草莓客户管理系统--主业/`。
- 每次对草莓客户管理系统做完实质开发、数据结构调整、UI 改版、测试补充或项目资料同步后，都要追加当天开发日志。
- 开发日志写入 `/Users/gd/Library/Mobile Documents/iCloud~md~obsidian/Documents/项目管理/草莓客户管理系统--主业/开发/YYYY-MM-DD.md`。
- 同步维护开发日志索引 `/Users/gd/Library/Mobile Documents/iCloud~md~obsidian/Documents/项目管理/草莓客户管理系统--主业/开发/开发日志索引.md`。
- 开发日志索引固定使用 `固定位置 / 记录规则 / 已建日志 / 后续约定` 结构，`已建日志` 用 Obsidian wiki 链接按日期递增排列。
- 日期文件正文不要写 `# YYYY-MM-DD`，Obsidian 顶部已经显示文件名。
- 每篇最上方先写 `## 今日概览`，用简单白话讲清楚今天做了什么。
- 记录内容保持务实：今日概览、关键记录、测试与验证、文档与资源、后续处理、本次采集情况。
- 不要把缓存、构建输出、临时文件和无关聊天写进开发日志。
- 当前系统虽然应用名仍是“草莓客户管理系统”，但项目数据已经同时承载客户项目和内部主业事项；内部主业事项固定使用 `草莓主业` 作为关联对象，不要创建 `客户--草莓主业.md`。
- 内部主业事项如果项目名未显式带月份前缀，默认按录入日期补成 `YYYY-MM 项目名`，保持总览和项目页口径一致。
- 首页 `客户总览` 只做本周跟进台，不再承载客户档案详情；不要把旧版 `客户库` 详情卡、相关项目详情或手动编辑客户入口重新塞回首页，全部客户沉淀统一去独立 `客户库` 页面。
- `草莓主业` 下的活跃内部项目即使暂时没有 `下一步` 或 `下次跟进日期`，只要已有 `当前重点`，也必须进入首页 `待排期`，避免内部主业事项只出现在项目管理里。
- 会话 `019dd811-0fab-7bc0-9112-e355b54a9182` 固定视为日常客户/主业进度入口：自动提醒、用户回复进展、客户/项目/内部主业事项回写都按这个上游语境衔接。
- 当前开发线程固定负责系统建设：App、数据结构、skill 规则、自动化口径和开发日志；不要把日常跟进回复直接沉淀到开发日志里。
- `customer-progress-followup` 负责每天读取客户/项目数据并生成追问；`客户进度` 负责根据用户回复写回客户页、项目页、总表、主业记录和必要的滴答关联。
- 自动化、录入助手、客户管理助手和 App 必须共享同一套 Obsidian 数据源，不新增平行提醒数据库，也不单独维护一份“提醒记录”。
- 如果用户给的是主业客户 / 品牌项目的真实进展，同时又明确提出 `记得`、`提醒`、`放滴答`、`建子任务`、`挂主任务`、`节后第一天记得` 这类时间管理动作，默认两边同步：既更新草莓客户管理系统，也创建或调整滴答任务；除非用户明确说只处理其中一边。
- 关系人数据固定写入 `/Users/gd/Library/Mobile Documents/iCloud~md~obsidian/Documents/项目管理/草莓客户管理系统--主业/人员数据/`，人员文件名直接使用人名，例如 `人员/客户对接人A.md`，不要再创建 `关系人--客户对接人A.md` 这类前缀文件。
- 用户纠正关系人的性别、联系方式、喜好、雷区或判断时，必须把人员页和 `00 关系人总表.md` 当作可编辑事实源更新；性别选项固定为 `男 / 女 / 待判断 / 不适用`，人工确认优先于 AI 猜测。
- 项目页正面参与人保持极简，只显示 `所属方 / 关系 / 人`；人的性格、喜好、联系方式和长期判断放到关系人库，不要塞回项目卡片。

- 桌面应用固定浅色模式；修改启动或主题时，不要移除 `app.setStyle("Fusion")`、全局浅色 palette、以及关键 QSS 防漏底选择器：`QAbstractScrollArea::viewport`、`QTabWidget::pane`、`QSplitter::handle`、`QToolTip`、`QComboBox QAbstractItemView`。
- 如果修改 `/Users/gd/Desktop/草莓客户管理系统.app/Contents/Info.plist`，必须保留 `NSRequiresAquaSystemAppearance = true`，并重新执行 `codesign --force --deep --sign - /Users/gd/Desktop/草莓客户管理系统.app`。
