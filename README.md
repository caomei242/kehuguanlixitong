# 草莓客户管理系统

本地 Mac 桌面主业工作台，从客户管理扩展到客户、项目和内部主业事项的轻量管理。当前主要读写本机和 Obsidian 里的 Markdown 数据，不提供云端同步、后台自动抓取或无人值守自动导入。

## 当前能力

- 客户总览：读取 `客户数据/` 和 `项目数据/` 下的 Markdown，把首页收成“本周跟进台”，只重点展示逾期、今天、明天、后续排期和待排期的客户/项目/内部主业节点；已归档和暂缓默认不进首页。首页不再嵌入客户库详情卡，全部客户档案回到独立 `客户库` 页面。顶部 `事项范围` 可在 `全部事项 / 客户相关 / 内部主业` 之间切换，默认看全部事项。
- 快速录入：把客户聊天、需求、推进记录或内部主业事项原文整理成结构化表单；AI 会先判断是 `客户更新 / 客户项目 / 主业事项`，保存前需要人工确认，再写入对应客户或项目资料。
- 截图识别：快速录入页支持粘贴截图、拖拽图片或选择图片，调用 MiniMax 识别并整理到录入表单；钉钉审批导入区也支持审批截图 OCR。
- 字段锁定：快速录入表单里的关键字段可以锁定，避免后续 AI 整理时覆盖已经确认过的客户名、类型、需求、下一步等内容。
- 项目管理：按客户或主业事项查看项目看板和项目详情，支持筛选客户、年份、状态；默认先看项目摘要，点 `查看项目` 再展开详情。品牌项目可从桌面主业项目目录同步，网店 KA、网店店群、博主和内部主业事项会使用不同进度流程，不再默认套 `审批 / 合同` 节点；`已归档` 项目默认排在看板底部，但仍可通过状态筛选单独查看；项目卡片会提示是否已关联滴答日记任务。
- 关系人库：集中沉淀内部负责人A、客户对接人A、客户对接人B、内部负责人B 这类主业关系人；人员页可手动编辑性别、所属方、组织、联系方式、判断、喜好和雷区，客户页和项目页只保留极简关系引用。
- 钉钉审批导入箱：项目管理页支持粘贴审批文字、拖入审批文件、扫描固定导入箱，并先预览归属再写入项目或客户的审批记录。
- 设置页：通过 `路径配置` 集中配置客户数据路径、项目数据路径、主业文件根路径、钉钉审批导入箱；通过 AI 配置维护 MiniMax API Key、模型和 Base URL。

## AI 主业参谋协作链路

- 日常入口固定串联会话 `019dd811-0fab-7bc0-9112-e355b54a9182`：它负责自动提醒、接收你的进展回复，并触发客户/项目/内部主业事项回写。
- `customer-progress-followup` 是每天的参谋脑子：只读取客户数据和项目数据，按 `昨天未变化事项 / 今天重点事项 / 明天重点事项 / 待排期事项` 生成行动清单。
- `客户进度` 是写回执行者：根据你的回复同步客户页、项目页、总表、主业记录和必要的 `## 滴答日记` 关联。
- 桌面 App 是工作台：展示本周跟进、项目进度、客户库和滴答关联状态，不单独维护提醒数据库，也不替代自动化参谋。
- 录入助手、客户管理助手和自动化都必须写同一套 Obsidian Markdown；不要另起一份提醒记录或平行数据库。

## 本地启动

在项目根目录运行：

```bash
PYTHONPATH=src python3 -m strawberry_customer_management.app
```

如果需要通过环境变量提供 MiniMax Key：

```bash
MINIMAX_API_KEY=你的_key PYTHONPATH=src python3 -m strawberry_customer_management.app
```

MiniMax API Key 也可以在设置页保存到本机用户配置。不要把 Key 写进仓库、README 或提交到 `.env`。

## 钉钉审批导入方式

项目管理页的钉钉审批导入区支持三种入口：

1. 复制钉钉审批列表或审批详情文字，粘贴后点击 `预览归属`。
2. 粘贴、拖拽或选择审批截图，识别完成后会把文字放到导入文本框，再预览归属。
3. 把 `xlsx / csv / txt / md / pdf` 文件拖入导入区，或放到导入箱后点击 `扫描导入箱`。

导入流程是先解析、再预览、最后人工点击 `写入审批`。系统会优先写入匹配到的项目；只匹配到客户时写入客户的待归属审批；归属不清时进入待人工确认口径。拖入文件只会复制到导入箱并生成预览，不会静默写入 Obsidian。

PDF 会优先用 `pypdf` 提取真文本。如果 PDF 是扫描图片，仍需要走截图 OCR 或人工确认。

## MiniMax 配置

- 默认模型：`MiniMax-M2.7`
- 中国大陆 Base URL：`https://api.minimaxi.com/v1`
- Global Base URL：`https://api.minimax.io/v1`
- 快速录入、客户截图识别、钉钉审批截图识别共用设置页里的 MiniMax 配置。
- 如果设置页没有填写 API Key，应用会读取环境变量 `MINIMAX_API_KEY`。

## 使用注意

- 写入客户和项目资料前，先检查客户名、项目名、当前需求、下一步和审批归属。
- 首页里的 `后续排期` 表示已经定了日期、但不在本周内的事项；`待排期` 只保留“有下一步但还没定日期”的事项。内部主业事项如果已经有 `当前重点`、但还没有 `下一步` 或 `下次跟进日期`，也要进入首页 `待排期`，避免只躲在项目管理里。
- 内部主业事项用于系统建设、流程优化、资料整理等非客户任务；它们归到 `项目数据/项目/草莓主业/`，不会生成 `客户--草莓主业.md`。
- 内部主业事项如果录入时没手写月份前缀，系统会按当前日期自动补成 `YYYY-MM 项目名`，保证它和客户项目在总览里的显示口径一致。
- 客户类型当前支持 `品牌客户`、`网店KA客户`、`网店店群客户`、`博主`，并且支持多选；如果一个对象既能推广新功能/新产品，又可能自己使用软件，不要二选一，客户类型直接写 `博主 / 网店店群客户` 这类并列身份，二级标签写 `小时达 / 微信 / AI商品图` 这类固定渠道或场景。客户类型和二级标签选项可在设置页维护。
- 项目类型要决定流程口径：内部主业走事项推进，网店 KA 走试用/开通/使用/增购，网店店群走点数/批量开通/续费，博主走合作/排期/发布/复盘；只有品牌合同项目才显示审批、合同和法务相关节点。
- 人员页文件名直接用人名，例如 `人员/客户对接人A.md`；如果你在关系人库里手动修了性别、联系方式、喜好或雷区，系统会写回人员页和关系人总表。AI 只能预填，人工确认的人员字段优先级更高。
- 字段锁定只影响快速录入页当前整理过程，不等于数据权限或版本锁。
- 钉钉审批导入不是后台自动同步；每次都需要手动粘贴、拖入文件或扫描导入箱，并确认预览后写入。
- 滴答日记目前只做项目关联状态展示，桌面应用本身不会自动创建、修改或完成滴答任务；真实任务仍由录入助手、`客户进度` 或滴答相关 skill 处理。如果用户明确给出主业客户 / 品牌项目进展，同时又要求提醒、子任务或节后记得，外部 assistant workflow 默认应把客户系统和滴答一起同步，除非用户说明只处理其中一边。
- 桌面项目同步来源是本机 `主业/品牌项目/` 目录，不会自动创建真实业务文件。

## 本地图片预览规则

- 当前默认预览目录：`/Users/gd/Desktop/customer-icon-options/`
